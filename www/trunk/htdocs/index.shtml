Welcome to OpenHPI's web site!

OpenHPI is an open source project created with the intent of providing an implementation of the SA Forum's Hardware Platform Interface (HPI). HPI provides an abstracted interface to managing computer hardware, typically for chassis and rack based servers. HPI includes a resource modeling; access to and control over sensor, control, watchdog, and inventory data associated with resources; abstracted System Event Log interfaces; hardware events and alerts; and a managed hotswap interface.

OpenHPI provides a modular mechanism for adding new hardware and device support easily. Many plugins exist in the OpenHPI source tree to provide access to various types of hardware. This includes, but is not limited to, IPMI based servers, Blade Center, and machines which export data via sysfs. For up to date status on these components, please check out the status page.

The OpenHPI project is always looking for more participation. If you are interested in using OpenHPI, or participating the evolution of OpenHPI, please join our Mailing List by following the links on the sidebar.

OpenHPI 0.6 Released! (03/22/2004)
New in this release:
  • Build System
    • Debug messages are now configurable (configure --enable-debug to turn them on).
    • Check for proper versions of Net-SNMP, OpenIPMI, and FAM.
    • Check for minimum supported GCC version (3.2.0)
    • CVS directory structure changed so plug-ins, core library, EP and RPT utils and clients are siblings.
    • HPI command line utilities moved from Utils directory is now known as clients.
  • Base Library
    • saHpiResourceIdGet supported
    • Domain System Event logs supported with configurable event severity threshold
    • Global configuration stanzas added to openhpi.conf
    • Entity Path and RPT utils are compiled as a seperate library (libopenhpiutils.so).
    • Core library is now 64bit clean, and compiles on IA64 systems.
    • New libopenhpi_snmp for common snmp calls.
  • Plugins
    • IPMI-DIRECT plug-in:
      • Fast discover process
      • Fail-safe RMCP connection
      • Framework for vendor specific features
      • Controls discovery added including FAN controls
      • Common base class for Sensors and Controls
    • OpenIPMI based IPMI plug-in
      • Requires OpenIPMI library 1.2.21 or later (strict)
      • Support for set/get time on Management Controllers with SEL capability
      • FRU/Inventory support added
      • Support for MC Event Log clearing
      • Enhanced Management Controllers discovery and detection of SEL
    • Blade Center plug-in
      • Event log cache created to speed up event operations (x100 speedup gained)
      • Support for time set/get operations
      • Support for event log clear
      • Enhanced event reporting
      • Various code clean ups
    • RSA (Remote Service Adapter) plug-in
      • First release of this plugin (snmp_rsa)
      • Resource discovery supported
    • SNMP Client plug-in
      • Support for HPI Subagent 0.2 added
      • Sensor tables supported
      • SNMP v3 authentication and encryption supported
  • HPI Daemon
    • First release of new HPI client/daemon prototype
    • libopenhpiclient.so created for using HPI client/daemon
  • Documentation
    • OpenHPI utility functions (libopenhpiutils.so) documented in OpenHPI Manual.
    • Configuration for snmp_rsa added to manual
  • Testing
    • New unit tests for RPT utils.
HPI Subagent 0.2 Released! (3/04/2004)
  • Second release of the HPI SNMP sub-agent.
  • Support for full implementation of the HPI-MIB v1.4 (included). This means HPI 1.0 instrumentation using HPI-MIB objects in twenty-three tables:
    • Resource Presence Table entities,
      • HotSwap capabilities,
    • Resource Data Records,
      • Sensors
        • Current sensor reading,
          • Reading values for
            • Max,
            • Min,
            • Nominal,
            • Normal maximum,
            • Normal minimum
          • Threshold values for:
            • Upper Critical,
            • Upper Major,
            • Upper Minor,
            • Lower Critical,
            • Lower Major,
            • Lower Minor,
            • Positive Hysteresis,
            • Negativie Hysteresis
      • Inventory,
      • Control,
      • Watchdog
    • Events and System Event Log
  • SGML Guide for writing a sub-agent using NetSNMP API, titled: "NetSNMP subagent development manual."
  • Support for notifications/TRAPS as defined in the MIB:
    • For Events:
      • Sensor,
      • Hot Swap,
      • Watchdog,
      • OEM,
      • User
    • For Resource Data Records,
    • For Resource Presence Table
  • Support for user configurable options:
    • update timer for new RPT/RDR/Events ,
    • maximum events held in eventw row before they are automatically removed,
    • sending of new found RPT/RDR/Events during startup.
  • Example code for retrieving instrumentation information using SNMP.
  • Fix many bugs.
